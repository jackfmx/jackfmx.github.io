模块化设计

模块化的思想和关注点的分离

包容变化是模块化的主要作用

基本模块化的写法

开-闭原则 对于扩展是开放的，对于修改是封闭的
模块复用和系统复用
分离业务逻辑层和数据存储层
将系统模块放在不同的源文件中

代码设计中的一些常见方法：

  • KISS （keep it simple & stupid） 一个函数或方法只做一件事
  • using design to frame the code 用设计来框代码 把设计转化成伪代码来描述整个系统的结构 including pseuducode 伪代码
  • 不要和陌生人说话原则
  • 合理利用Control Structures Data Structures 来简化代码
  • 一定要有错误处理

接口

接口定义了软件单元对系统其他部分提供的服务
函数名、参数、返回值是接口的三个显性要素
前置条件和质量属性是接口的两个隐性要素

接口的五要素

  • Purpose 接口的目标
  • Precondition(assumption) 前置条件 (假设的条件)
  • Protocols 参数、返回值类型、指针指向的数据格式
  • Postconditions (visible effects)
  • Quality attributes eg. 执行时间

接口的分类

  • 共享数据或变量名
  • call-in functions
  • callback functions
  • 同步调用接口
  • 异步调用接口

接口与模块之间的耦合程度的关系 高内聚低耦合

Common Coupling 公共耦合 - 共享数据区或变量名
Stamp Coupling 标记耦合 - 复杂的数据结构在模块之间传递
Data Coupling 数据耦合 - 在模块之间只传递数据

Generality ： Parameterizing Context，Removing preconditions， Simplifying postconditions

通用接口设计 ： 参数化上下文，移除前置条件，简化后置条件

###callback函数

B函数指针作为A函数的参数传入A函数，当A函数运行时，通过函数指针找到B函数，从而执行B函数，B函数就是一个回调函数

回调函数的作用是可以写出更加通用的程序，实现代码复用。一些通用逻辑写在A函数，而把一些具体的实现写在B函数，这样执行通用A函数的时候可以通过函数指针找到并运行B函数，实现了对A函数代码的复用。如果没有回调函数，那在执行A函数的时候只能执行一些固定的不可改变的逻辑，从而不能实现函数代码的复用

C语言的库函数qsort函数中的参数cmp就是一个回调函数。qsort是一个通用的算法，而cmp函数包含了对具体的值的比较逻辑。Java中的事件也用到了回调函数

函数的可重入性

可重入函数可以由多于一个任务并发使用，而不必担心数据错误。
不克重入函数不能由超过一个任务所共享，除非能确保函数的互斥（或者使用信号量，或者在代码的关键部分禁用中断）
可重入函数要么使用本地变量，要么在使用全局变量时保护自己的数据

可重入函数：

  • 不为连续的调用持有静态数据。
  • 不返回指向静态数据的指针；所有数据由函数的调用者提供
  • 使用本地数据，或者通过制作全局数据的本地拷贝来保护全局数据。
  • 绝不会调用任何不可重入函数

线程的关键特点是指令的执行流

线程安全问题： 是由全局变量及静态变量而引起的。若每个线程中对于全局变量和静态变量只有读操作，而无写操作，一定是线程安全的；若有多个线程同时执行读写操作，一般都需要考虑线程同步，否则有可能影响线程安全。

函数的可重入性与线程安全之间的关系

  • 可重入函数不一定是线程安全的
  • 可重入函数在多个线程中并发使用时是线程安全的，但不同的可重入函数（共享全局变量及静态变量）在多个线程中并发使用时会有线程安全问题（可能是线程安全也可能不是线程安全的）
  • 不可重入的函数一定不是线程安全的

不同的质量视角

  • 产品的视角（内部质量）
  • 用户的视角 – 是不是对用户有帮助
  • 商业上下文的质量 – 投资回报和驱动力问题
  • 运营推广
  • 开源

设计原则 分解系统功能和系统行为

  • modularity模块化
  • interfaces接口
  • information hiding信息隐藏
  • incremental development增量开发
  • abstraction抽象
  • generality一般化

重构的目标是简化方案或优化设计
信息的隐藏是由接口定义来达成的
信息的隐藏可以提高软件的可维护性，即包容变化

  • 隐藏数据的表现形式 – 信息内聚
  • 隐藏算法 – 功能内聚

增量开发需要以模块化设计为前提条件
